# PRD：徐涛考研政治“题前背”模块需求文档

**版本：** V1.1

**状态：** 评审准备中

**产品经理：** Gemini (AI 协作)

**模块定义：** 政治考研“死概念”背诵与 AI 对话测评闭环。

---

## 1. 业务目标与逻辑

通过“听（磨耳朵）- 读（看卡片）- 说（AI测评）”三位一体的方式，帮助学员利用碎片化时间攻克政治大纲中的核心死概念。

### 核心业务流

1. **输入阶段**：用户在大纲列表选择章节，通过文字卡片和平铺音频进行记忆。
2. **输出阶段**：进入对话框，AI（徐涛）语音提问，用户语音或文字回答。
3. **反馈阶段**：系统根据关键词命中率打分，用户决定“重测”或“下一题”。

### 2.4 人工修正交互 (Manual Correction)
*   **场景**：当 ASR 识别错误导致评分偏低时（例如方言、同音词误判）。
*   **操作**：用户点击语音气泡旁的“编辑/铅笔”图标。
*   **反馈**：气泡转为文本输入框，用户修改文字后点击“提交”，系统基于新文本重新打分。

## 3. 核心功能逻辑与算法规则

### 3.1 打分算法 (Matching Logic)

* **处理流**：语音输入 -> ASR转文字 -> 文本清洗（去语气词） -> 关键词匹配。
* **计算公式**：$Score = \frac{命中核心关键词数量}{该考点总预设关键词数量} \times 100$。
* **容错机制**：建立同义词库（例如：“决定性”与“决定”、“群众”与“人民群众”在特定题目下视为等同命中）。

### 3.2 徐涛 AI 反馈机制

* **提问**：调用徐涛 AI 音频。
* **反馈话术**：
* **满分**：“太棒了，完全正确！这就是满分卷，点个赞！”
* **未满分**：“这题你拿了 {X}% 的分！🎉 命中：{A}，缺失：{B}。再接再厉！”


* **决策动作**：
* 点击“重新回答”：系统通过AI气泡回复“好的，请重新回答：[原题目内容]”，用户重新录入。
* 点击“跳过/下一题”：
    *   若为普通题目：直接进入下一题。
    *   若为最后一道题：按钮文案变为“🚫 这是最后一道题，无法跳过哦”，点击弹出提示，强制完成。
* 点击“完成挑战”：当且仅当所有题目（包括最后一道）回答完毕后出现，点击返回章节页。

## 4. 技术与性能要求 (Non-functional)

1. **语音交互**：录音时长限制（最小 1s，最大 60s）。
2. **转写速度**：ASR 识别延迟不得超过 1.5s，需有“识别中”的状态反馈。
3. **音频体验**：
   *   **预加载**：由于“磨耳朵”功能是高频场景，必须实现音频文件的预加载机制（当前播放第 N 句时，后台静默下载第 N+1, N+2 句），确保切题无卡顿。
   *   **音源策略**：系统应支持“真人录音”与“TTS 合成”两种模式的无缝切换。MVP 阶段优先使用真人录音，但架构上需预留 TTS 接口。
4. **数据埋点**：记录每章的测试通过率、重测率、人工修正率（作为识别优化依据）。
